Web Project (Servlet/Jsp를 이용한 웹사이트 구축)

프로젝트명 :

개발자들을 위한 커뮤니티(All for Developer/A4D)


- 사용자(일반,기업 사용자)에게 필요한 정보나 서비스를 제공하는 커뮤니티 페이지입니다.

- 일반 사용자가 게시글을 통해 소통하고 스터디, 채용공고, 공간대여 서비스를 이용할 수 있고

  기업 사용자는 구인공고를 이용할 수 있습니다.


개발환경 및 순서


- 개발기간: 2021/07/19 ~ 2021/08/06

- 개발 툴: SQL Developer, eclipse, erdCloud

- 사용언어: Java, sql 언어, HTML, CSS, JavaScript

- 순서
  : 기획서 > 요구분석서 > 기능명세서 > ERD,DB > 화면설계서 > 소스 > PPT(발표) > Java Doc > 최종요약본



구현 목표



- 일반 사용자가 게시글로 다른 사용자들과 소통하도록 한다.

- 일반 사용자가 커뮤니티,스터디, 채용공고, 공간대여 서비스를 이용할 수 있도록 한다.

- 기업 사용자가 구인 공고를 등록하도록 하여 구직자를 직접 받을 수 있도록 한다.


담당업무 구현화면 및 설명



- ERD Model


- 기획서, 요구분석서, UI설계 후 DB작업에 들어가면서 Query와 ERD를 구현하였습니다.

- 테이블들을 생성하고 더미들을 insert하였습니다.

- 구현한 ERD를 보고 프로젝트 기능들을 구상했습니다.

- 회원가입


- 사용자가 사이트를 사용하기 위해 계정을 만들수 있는 회원가입을 구현했습니다.

- 사용자가 회원가입을 하면 tblmember로 데이터가 insert됩니다.

- 회원가입시 기존에 있는 ID/닉네임은 중복되지 않게 조건을 걸어놨고

  다른 요소들은 유효성 검사를 하여 올바른 값이 들어가게했고 카카오API를 사용해 주소를 검색가능하게했습니다.

- 사용기술 : ArrayList, JSTL, EL, Statement, ResultSet, 카카오 API


- Login


- 회원가입이 완료된 데이터에서 id와 pw가 일치한다면 세션을 저장해줍니다.

- 쿠키를 이용해 아이디 저장하기를 체크해두면 100일동안 아이디각 저장되게 구현했습니다.

- 사용기술 : ArrayList, JSTL, EL, Statement, ResultSet, Session


- 게시글 목록


- 일반사용자는 커뮤니티 카테고리에서 필터 기능을 통해 원하는 게시글들의 목록을 볼 수 있습니다.

- 많은 양의 게시물들을 페이징을 통해서 한페이지에 10개의 게시글을 볼 수 있게 구현하였고

  사용하고있는 view에 (현재시간 - 작성한시간)의 컬럼을 추가해 24시간보다 작을 경우 옆에 new라고 나타납니다.

- 로그인 완료된 상태여야지만 글쓰기가 가능하게 구현하였습니다.

- 사용기술 :JSTL, EL, Paging, Statement, ResultSet, Session, Calendar, HashMap


- 게시글 목록 검색하기


- 커뮤니티 목록에서 사용자는 원하는 단어를 검색해서 관련 게시글을 볼 수 있습니다.

- 검색하는 상황일때의 조건을 따로 걸어 해당 단어가 포함돼 있는 게시글들의 목록을 나타내줍니다.

- 해당 단어를 확실히 알아볼 수 있게 background-color를 걸어주었습니다.

- 사용기술 :JSTL, EL, Paging, Statement, ResultSet, Session, Calendar, HashMap


- 게시글 상세페이지


- 게시글의 내용을 확인할 수 있고 로그인이 된 상태이면 추천, 스크랩, 댓글작성도 가능합니다.

- 자신이 작성한 게시글이나 댓글은 수정,삭제가 가능하고 한개의 아이디로 조회수를 계속 늘리지 않기 위해서
  사상세페이지에 들어가면 read세션을 n으로 변경하였습니다. 로그인이 안되어있는 경우,
  위의 두번째 사진처럼 로그인을 클릭하면로그인 페이지로 이동하게 구현하였습니다.

- 사용기술 : JSTL, EL, PreparedStatement, ResultSet, Session, Calendar, HashMap


- 게시글 작성하기


- 로그인한 상태에서 글쓰기 버튼을 누른 후 자신이 글쓰기를 원하는 카테고리를 고른후
  제목과 내용을 적은 후 등록하기 버튼을 누르면 작성한 데이터들이 해당 테이블로 insert되고
  해당 카테고리의 게시글 목록에 포함되게 구현했습니다.

사용기술 : JSTL, EL, PreparedStatement, ResultSet, Session, Calendar, HashMap


- 게시글 수정하기


- 자신이 작성한 게시판의 글을 수정하기 위해서 가지고 있는 데이터에서
  글번호가 같은 게시글의 데이터를 모두 가져와 내용을 수정할 수 있는 페이지를 보여줍니다.

- 내용을 모두 수정한 후 수정하기 버튼을 누르면 데이터가 수정된 내용으로 업데이트가 됩니다.

- 사용기술 : JSTL, EL, PreparedStatement, ResultSet, Session, Calendar, HashMap


- 게시글 삭제하기


- 자신이 작성한 게시글 삭제하기 버튼을 누르면 게시글 삭제 페이지를 보여줍니다.

- 삭제하기 버튼을 누르면 해당 게시글이 Foreign Key로 걸려있는 게시글의
  스크랩,추천,댓글의 데이터를 먼저 delete 해준 후 고유번호의 데이터를 delete합니다.

-해당 게시글 데이터가 delete 때문에 목록으로 이동하면 게시글이 사라지게됩니다.

- 사용기술 : JSTL, EL, PreparedStatement, ResultSet, Session, Calendar, HashMap


- 게시글 차트보기


- 게시글 카테고리별로 몇개의 글이 있는지 차트를 볼 수 있습니다.

- 커뮤니티 게시글 데이터로 View를 만들어서 카테고리별로 갯수를 count한 값을 가져옵니다.

-만들어진 View와 Chart API를 사용해 차트를 구현했습니다.

- 사용기술 : JSTL, EL, PreparedStatement, ResultSet, Session, Calendar, HashMap,High-Chart


- 마이페이지 내가쓴게시글


- 회원이 커뮤니티 게시글에서 스크랩한 게시글을 마이페이지에서 확인할 수 있습니다.

- 스크랩한 게시글들의 목록을 누르게되면 해당 커뮤니티 게시글 페이지로 이동하게됩니다.

- 사용기술 : JSTL, EL, PreparedStatement, ResultSet, Session, Calendar, HashMap


프로젝트를 통해 느낀점


- 웹 프로젝트를 진행하면서 웹서비스 구상, 기획, 개발의 전체 프로세스를
  통해서 웹이 어떻게 개발되는지를 실현할 수 있었고 페이지 한장당 많은
  노력이 필요하다는것을 몸소 느낄 수 있었습니다. 자바 프로그램 안에서 데이터베이스의
  연결을 가능하게해 이벤트가 일어났을때 데이터를 바로 변경할 수 있다는것이
  흥미로웠습니다. 프로젝트 초반에는 구상하기도 힘들고 오류가 날때마다 오류 찾기가 힘들었지만
  프로젝트를 진행하는 동안 같은 오류가 뜨는 경우도 있었고 오류메세지를 보고 구글링도 하고
  오류가 난 부분을 찾아서 수정하면서 프로젝트 마무리 단계에서는 오류를 보다 신속하게 찾고 수정할 수 있게 발전했습니다.
  프로젝트를 마무리하면서 일상생활에서 쓰는 웹페이지들도 이러한 과정으로 만들어진다는것을
  알았고 프로젝트를 마치면서 뿌듯함을 느낄 수 있었습니다.
